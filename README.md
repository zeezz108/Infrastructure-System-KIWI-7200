## О проекте

Система предназначена для автоматизированной диагностики IT-инфраструктуры. Агенты, установленные на целевых машинах, получают задачи от центрального сервера, выполняют проверки (информация о хосте, сетевые данные, открытые порты, обнаружение сервисов) и возвращают результаты. Сервер хранит историю, предоставляет веб-интерфейс для управления и визуализации.
Централизованная система для диагностики инфраструктуры. Агенты получают задачи от сервера, выполняют проверки и возвращают результаты.

### 1. Запуск сервера
```bash
python backend/main_simple.py
```

Сервер запустится на http://localhost:8000

Логин: admin

Пароль: admin123

### 2. Добавление агента через веб
Открой http://localhost:8000/agents.
Нажми "+ Добавить агента".
Введи имя.
Нажми "Зарегистрировать".

### 3. Запуск агента
Открой http://localhost:8000/agents.

1 вариант (через Web):
Нажми "Запустить".

2 вариант (через Terminal): Нажми "Команда" напротив нужного агента, затем "Копировать". 
Вставь команду в Terminal и запусти.

### 4. Создание задачи
На странице агентов выбери агента.
Выбери тип проверки.
Нажми "Запустить".

### 5. Функционал
1) Веб-управление агентов: Регистрация/Запуск/Остановка/Удаление агентов через веб
2) Очередь задач
3) Шаблоны проверок 
4) Экспорт задач в JSON/CSV 
5) Хранение истории в SQLite 
6) Сравнение результатов задач 

### 6. Киллерфичи

- Визуализация связей — интерактивный граф доступности между агентами и сервисами
- Рейтинг агентов — оценка стабильности и скорости выполнения задач
- Real-time обновление — статусы и метрики обновляются автоматически без перезагрузки страницы
- Ограничение количества одновременных задач на агенте (2 задачи)
- Метрики и графики
- Группы агентов

### 9. Сценарии использования

Сценарий 1: Базовая диагностика нового сервера
1. Добавить и запустить агента через веб
3. Выбрать шаблон "Базовая диагностика"
4. Получить: hostname, IP, ОС, открытые порты

Сценарий 2: Проверка доступности сервисов
1. Запустить шаблон "Проверка сервисов"
2. Агент проверяет HTTP, HTTPS, SSH, MySQL, PostgreSQL
3. В результатах видно какие сервисы доступны

Сценарий 3: Экспорт отчёта
1. Перейти на страницу задач
2. Нажать "Экспорт CSV"
3. Открыть файл в Excel для анализа
